A · New candidate posted in group
1. Worker sees message → POST /api/ingest (text + photo + msgId)
2. Vercel: photo→Blob, Haiku parses Hebrew → creates active candidate
3. Bot replies to that message: «נוסף ✅ דנה, 27, מורה — לפרטים: app.../c/abc» + saves confirmation-msgId ↔ candidate
B · Correction (reply to the bot's confirmation)
4. Someone quote-replies the bot: «הגיל לא נכון, היא בת 28»
5. Worker matches quoted msgId → POST /api/correct → Haiku reads the change → updates candidate + writes audit log
6. Bot replies: «עודכן ✅ גיל: 28»
No pre-approval gate — corrections + full edit history are the safety net. Matchmakers also edit anything directly in the web app.
